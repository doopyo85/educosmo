개요
================================================================================

Educosmo 프로젝트는 3가지 독립적인 레이아웃 시스템으로 구성되어 있습니다.
각 시스템은 서로 다른 목적과 구조를 가지며, 상호 간섭 없이 독립적으로 유지됩니다.

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  1. COMPONENT-SYSTEM  → Python, Algorithm, Certification               │
│     - 고도로 모듈화된 컴포넌트 기반 학습 시스템                        │
│     - IDE, Quiz, Jupyter 등 동적 컴포넌트 로딩                         │
│                                                                         │
│  2. CARD-LAYOUT-SYSTEM → Index, Scratch/Entry Project                  │
│     - 간단한 카드 기반 프로젝트 선택 UI                                │
│     - Bootstrap Carousel + Grid 레이아웃                               │
│                                                                         │
│  3. BOARD-SYSTEM → Teacher, Board, S3 Browser                          │
│     - 고정 사이드바 + 메인 콘텐츠 레이아웃                             │
│     - 관리자/교사용 대시보드 및 게시판                                 │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

================================================================================
시스템 경계 (System Boundaries)
================================================================================

각 시스템은 명확히 분리된 파일과 디렉토리를 가지고 있습니다.

┌──────────────────────┬─────────────────────┬──────────────────────┐
│  Component-System    │  Card-Layout        │  Board-System        │
├──────────────────────┼─────────────────────┼──────────────────────┤
│ template.ejs         │ index.ejs           │ teacher.ejs          │
│ common-layout.css    │ card_project.css    │ board-layout.css     │
│ component-system.js  │ (Bootstrap only)    │ teacher.css          │
│ /css/components/     │ scratch_project.ejs │ board.css            │
│ /css/ide/            │ entry_project.ejs   │ /teacher/*.ejs       │
│                      │                     │ /board/*.ejs         │
└──────────────────────┴─────────────────────┴──────────────────────┘

⚠️ 주의사항:
- 한 시스템의 CSS/JS를 수정할 때 다른 시스템에 영향이 없는지 확인 필요
- 공유 파일: header.css, styles.css (기본 스타일만 포함)
- 각 시스템은 독립적으로 유지보수 가능

================================================================================
라우트 매핑 (Route Mappings)
================================================================================

Component-System Routes:
  /python_project      → Python 학습 페이지
  /algorithm           → 알고리즘 학습 페이지
  /certification       → 자격증 학습 페이지
  /pythontest          → Python 테스트 페이지

Card-Layout-System Routes:
  /                    → 메인 랜딩 페이지
  /scratch_project     → Scratch 프로젝트 선택
  /entry_project       → Entry 프로젝트 선택

Board-System Routes:
  /teacher/*                              → 교사 대시보드
  /teacher/student-management/progress    → 학생 진도 관리
  /teacher/student-management/list        → 학생 목록
  /teacher/student-management/attendance  → 출석 관리
  /teacher/class-materials/lessons        → 수업 자료
  /teacher/career-info/*                  → 진로 정보
  /board                                  → 누구리 보드
  /board/board_index                      → 게시판 목록
  /s3/student-files                       → 학생 파일 관리

================================================================================
세부 문서 참조
================================================================================

각 시스템에 대한 자세한 설명은 아래 문서를 참조하세요:

1. Component-System (컴포넌트 시스템)
   → component-system.txt
   - 컴포넌트 아키텍처 상세 설명
   - IDE, Quiz, Jupyter 통합 방법
   - CSS 변수 및 레이아웃 제어

2. Card-Layout-System (카드 레이아웃 시스템)
   → card-layout-system.txt
   - 카드 기반 UI 구조
   - Carousel 설정 및 반응형 디자인
   - 프로젝트 선택 페이지 가이드

3. Board-System (보드 시스템)
   → board-system.txt
   - Sticky 사이드바 레이아웃 가이드
   - Teacher/Board 페이지 구조
   - 새 페이지 추가 방법

================================================================================
공유 파일 (Shared Files)
================================================================================

다음 파일들은 모든 시스템에서 공유됩니다:

1. /public/css/styles.css
   - 전역 스타일 (폰트, 색상, 유틸리티)
   - ⚠️ 특정 시스템 전용 스타일 추가 금지

2. /public/css/header.css
   - 공통 헤더 스타일 (70px 고정 높이)
   - 모든 시스템에서 동일한 헤더 사용

3. /views/partials/header.ejs
   - 공통 헤더 HTML 구조
   - 로그인 상태, 네비게이션 메뉴

4. /views/partials/footer.ejs
   - 공통 푸터 (일부 페이지만 사용)

⚠️ 이 파일들을 수정할 때는 3가지 시스템 모두에 영향이 있으므로 주의!

================================================================================
개발 가이드라인
================================================================================

새 기능 추가 시:
1. 어느 시스템에 속하는지 먼저 결정
2. 해당 시스템의 세부 문서 참조
3. 다른 시스템에 영향 없는지 확인
4. CSS/JS 파일은 시스템별 디렉토리에 배치

수정 시 주의사항:
- Component-System: common-layout.css 수정 시 IDE 레이아웃 깨질 수 있음
- Card-Layout: styles.css의 카드 스타일 수정 금지
- Board-System: board-layout.css는 Board 시스템 전용

CSS 우선순위:
1. 시스템별 CSS (component/card/board)
2. 페이지별 CSS (teacher.css, board.css 등)
3. 공유 CSS (styles.css, header.css)
